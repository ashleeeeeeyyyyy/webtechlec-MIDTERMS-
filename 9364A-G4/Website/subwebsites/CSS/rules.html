CSS Rules

  • By origin and importance
  • By Specify
  • By Order
  • Regular Rules - These rules follow regular syntax
    • @charset - This defines the character set which is used by the web browser. This is very useful when the stylesheet contains non-ASCII characters.
    • @import - This rule is placed at the very beggining of the css file which instructs the stylesheet to include an external CSS files as if the requested file's content were right there.
    • @namespace - This is used particularly for applying CSS to XHTML so that the XHTML elements can possibly be used as selectors in the actual CSS file
  • Nested Rules- This contains subsets of additional statements
    • @document - This is used for specific conditions of styles which apply to a specified page. (e.g. Providing a URL then customizing styles only for the specified page.)
    • @font-face - This allows the user to use custom fonts for a webpage. Although some fonts may not be supported by other browsers, this rule will still accept the statements which create the custom fonts..
    • @keyframes - This rule serves as the basis for keyframe animations, which allow the web developers to mark the start and stop time for the animations.
    • @media - This contains plenty of additional statements for specifying styles to a specific screen.
    • @page - This defines styles used by individual pages when the document is being printed. Specifically, it contains pseudo-elements fo styling the very first page as well as the left and right margins of the page.
    • @supports - This test if the browser can support a certain feature, which then applies the stle for those elements if it is supported.

  • Properties
    • Shorthand properties
      • Allows authors to specify the values of several properties with a single property.
    • Vendor specific extensions (a.k.a, vendor prefixes)
      • Used by browser vendor as prefix for the names of experimental nonstandard CSS property.
        • style.css
    • Custom properties a.k.a CSS variables(Experimental)
      • index.html 
                          <p class = "first">The color is <b>black</b></p>
      style.css 
                          :root {
                        --color: black;
                        --color-blue: blue;    
                        --fontsize: 35px;    
                    }

                    b{
                        background-color: var(--color);
                        color: white;
                    }

                    .first{
                        font-size: var(--fontsize);
                        color: var(--color-blue);
                    }
    • Values
      • Reference: https://www.w3.org/TR/css-values-3/#component-types
      • Value types

        • Value types are designed in several ways

        • Keywords
      • Numbers
        • Dimensions
          • Length Units
            • Font-relative
              • Examples: em, ex, ch, rem
            • Viewport-percentage
              • Examples: vw, vh, vmin, vmax units
            • Absolute Lengths
              • Examples: cm, mm, q, in, pt, pc, px units
          • Angle Units
            • Usually used in transition and gradient.
            • These are the deg, grad, rad and turn units
          • Duration Units
            • Usually used in animation and transitions.
            • These are the s and ms units
          • Frequency Units
            • These are the Hz, kHz
          • Resolution Units
            • Usually used in media queries.
            • These are the dpi, dpcm and dppx.
        • Percentages
